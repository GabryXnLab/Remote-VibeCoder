# Comando: /plan

Quando invocato, prima di scrivere qualsiasi codice:

1. Leggi i file rilevanti per il task richiesto (usa @ per riferimenti espliciti)
2. Scrivi un piano dettagliato in `docs/PLAN.md` con:
   - Obiettivo del task
   - File che verranno modificati/creati
   - Step numerati da eseguire
   - Potenziali rischi o dipendenze
3. Mostra il piano all'utente e attendi conferma esplicita prima di procedere
4. NON scrivere codice finché l'utente non approva il piano
