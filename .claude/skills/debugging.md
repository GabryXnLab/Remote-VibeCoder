# Skill: Debugging

Quando invocato per un bug o errore:

1. Leggi SOLO i file direttamente coinvolti nell'errore (usa stack trace se disponibile)
2. Riproduci mentalmente il flusso di esecuzione
3. Formula massimo 3 ipotesi ordinate per probabilità
4. Verifica la più probabile prima di procedere con le altre
5. Prima di modificare: mostra la causa identificata e la fix proposta
6. Applica la fix minima necessaria — non refactoring non richiesto
7. Suggerisci un test per verificare che il bug sia risolto
