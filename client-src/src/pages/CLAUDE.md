# src/pages/ — Route-level page components

Three pages, each with a paired `.module.css`. Pages are composed of hooks + components; they own no business logic directly.

## `LoginPage.tsx`
Simple form → `POST /api/auth/login` → redirect to `/projects`. No session state needed; auth check on mount redirects if already logged in.

## `ProjectsPage.tsx`

Shows GitHub repos + active tmux sessions. All action handlers (clone, pull, push, open, kill, conflict resolution) are in `useProjectsActions` hook — the page focuses on rendering.

**Rendering pattern:** `repoActions(repo)` returns different button sets based on: repo.archived → repo.cloned + hasSession → repo.cloned → not cloned. Render helpers (`renderSyncIndicator`, `repoActions`) are plain functions inside the component, not `useCallback`, because they only re-run on render which is already gated by `repos` changing.

**Session section:** Shown above repos. Sessions without a `repo` association (free shell sessions) can appear here with no associated repo card.

## `TerminalPage.tsx`

The most complex page. Manages:
- Active session ID (switched via sidebar or URL param `?session=`)
- Multiple simultaneous terminal instances (rendered hidden when not active)
- Mobile vs desktop layout branching
- Space-hold mic, textarea fallback input, streaming banners

**URL param handling:** `?session=` is the primary way to load a specific session. The legacy `?repo=` param triggers `POST /api/sessions/:repo` and then redirects to `?session=` — this path still exists for backward compat with links generated by old code.

**Settings sections:** The settings panel is built inline in TerminalPage as `settingsSections` (array of `{ title, content: JSX }`). This JSX uses page-specific styles and state, making extraction to a separate component non-trivial. The streaming settings state is in `useStreamingSettings` hook; the rest of the state is page-local.

**Streaming banner:** A banner appears at the top of the terminal area when `activeStreamState === 'warn'` or `'suspended'`. On mobile it's a sticky strip; on desktop it's a centered overlay. The terminal itself keeps running behind the banner.

**Window manager (desktop):** `WindowManager` component renders all sessions as floating/tiled windows. On mobile, sessions are stacked divs with `display: none` for inactive ones.
